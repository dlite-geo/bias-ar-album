# 최애 AR 앨범 — 개발 인수인계 문서

> 이 문서는 AI 어시스턴트(또는 새 개발자)가 이 프로젝트를 이어받아 개발할 때 필요한 전체 맥락입니다.
> 원본은 [aivsomkar/PinViz](https://github.com/aivsomkar/PinViz) 를 포크해 대폭 개조한 것입니다.

## 프로젝트 개요

브라우저에서 완전히 로컬로 동작하는 3D AR 포토 갤러리.
사진을 드롭하면 화면 중앙(얼굴 자리)을 비운 **링(도넛) 배치**로 떠오르고,
웹캠 손 추적(MediaPipe HandLandmarker)으로 아이언맨 홀로그램처럼 잡기/던지기/회전/줌을 한다.
서버·DB·계정 없음. 사진은 기기 밖으로 나가지 않는다.

- 스택: React 19 + Vite + TypeScript, Three.js(커스텀 컴포저), Zustand, MediaPipe tasks-vision
- 실행: `npm install` → `npm run dev` (localhost:5173), 테스트: `npm test` (vitest, 37개)
- UI 언어: 전부 한국어

## 절대 되돌리면 안 되는 핵심 설계 결정 (전부 실제 버그를 고친 것)

1. **핀치 판정은 손바닥 크기 대비 비율** (`isPinchingScaled`, 잡기 0.5 / 놓기 0.9).
   절대 거리로 바꾸면 손이 카메라에 가까워질 때 홀드가 풀린다. 히스테리시스(잡기 빡빡, 놓기 느슨)와
   8프레임 추적 끊김 유예도 홀드 안정성의 핵심.
2. **`<video>` 요소는 모든 배경 모드에서 DOM에 붙어 있어야 한다** (블랙/화이트는 visibility:hidden만).
   DOM에서 떼면 브라우저가 미디어를 자동 일시정지시켜 손 추적이 통째로 죽는다.
3. **배치 경계는 구(sphere)가 아니라 카메라 시야 원뿔(cone)** (`clampToView`).
   회전은 시야축(z)만 사용 — z회전은 원뿔 안의 점을 원뿔 안에 유지하므로 사진이 화면 밖으로 못 나간다.
   x/y 회전을 자유롭게 풀면 이 보장이 깨진다 (🤌 기울이기는 ±0.6rad로 제한해 둠).
4. **MediaPipe 손 라벨은 신뢰 불가**: 거울 영상에서 양손이 같은 라벨로 나오거나(→화면 x좌표로 재배정),
   한 손이 두 개로 중복 감지된다(→중심 거리 <0.12면 하나 버림). `handTracking.ts`에 있음.
5. **잡기는 3프레임 지연 확정** (`pendingGrab`): 🤌를 만드는 과정에서 엄지+검지가 먼저 닿아
   가짜 핀치가 발생 → 3프레임 안에 🤌로 발전하면 잡기 취소.
6. **양손 스트레치는 "우세 동작 잠금"**: 크기(거리)와 회전(각도)이 같은 두 점에서 나오므로,
   먼저 임계값을 넘은 쪽 하나만 그 세션 동안 적용.
7. **양손 모드 진입 조건**: 홀드 중 반대 손이 "새로 핀치를 시작"했을 때만(두손 핀치),
   또는 손가락 4개를 활짝 편 손(편손 리사이즈). 노는 손의 어중간한 핀치가 끼어들면 이동이 망가진다.
8. **미러 좌표계(x반전+y다운) 때문에 회전 부호는 전부 반전**되어 있음. 손목 롤, 🤌 롤 등.

## 제스처 → 동작 매핑 (현재 확정 스펙)

- 🤏 핀치(잡기 지점=검지 끝, 커서 링 표시): 사진 잡기. 겹치면 맨 앞 사진. 잡히면 **빨간** 아웃라인,
  뒤에 가려진 후보는 **노란** 아웃라인 (OutlinePass 2개).
- 잡은 채 이동: 잡은 깊이 유지하며 따라옴. 손을 카메라 쪽으로: 그 사진만 가까이(팔 굽힘=손바닥 크기 비율,
  손목 롤 중엔 깊이 동결). 최소 거리는 사진 크기 기반(화면 넘침 방지).
- 잡은 채 휙 놓기: 던지기 — 관성으로 날아가 원뿔 벽에 튕기고 멈춘 자리가 새 정착지.
- 양손 핀치 스트레치: 벌리면 크기 / 돌리면 회전 (우세 잠금).
- 손목 롤(한손 홀드): 사진 회전.
- 🤲 양손 활짝 펴고 벌리기/모으기: 공간 줌. 관성 있음. 최대 확대(0.35×)로 화면 밖 연출 후
  0.5초 뒤 자동 복귀(`overshootPending`). 마우스 휠 줌은 자동 복귀 없음.
- 👋 스와이프: 좌우=180°, 상하=±90° 단발 회전, 속도 비례 빠르기, 0.6초 쿨다운.
  줌 직후 15프레임은 무시(한 손 추적 이탈로 인한 오발동 방지).
- 🤌 다섯 손가락 모으기(손 하나만 보일 때): **보라색** 하이라이트 + 공간 회전 노브.
  손목 롤 1:1, 손 이동으로 기울이기(±0.6rad). 3프레임 발동 / 10프레임 해제 유예 / lerp 스무딩.
- ✊ 주먹: 모든 공간 운동 정지.
- 손을 들어도 사진이 도망가진 않음. 손 모드에서는 잡기 판정만 강조하고, 사진 부유는 유지된다.

## 스노우볼 물리

- 각 사진은 홈 포지션 주변을 사인파로 부유(`driftParams`, xy는 작게 z는 크게).
- 링 전체가 z축으로 항상 천천히 회전(`IDLE_SPIN`).
- 놓은 자리 = 새 홈. `driftEase`가 0→1로 페이드되어 툭 튀지 않음.
- 버튼: ⊕중앙 모으기(사진 위치만) / ⟲완전 초기화(사진 위치·크기·회전·시점 전부) / ✋ 손 / 🖱 마우스·트랙패드(통합) / 얼굴 이모지 선택.

## 알려진 미세 이슈 / 다음 개선 후보

- 실사 손 누끼는 골격 스트로크 마스크(볼록껍질 아님)라 손가락 사이는 깨끗하지만
  손 윤곽이 두께 추정에 의존 — 세그멘테이션 모델로 업그레이드 여지.
- 얼굴 이모지는 FaceEmojiLayer + `faceEmojiGlyph`로 선택 가능. 손이 얼굴을 가리면 이전 위치를 유지하도록 보정.
- 레이아웃 persist는 세션 메모리뿐(새로고침하면 초기화). localStorage/IndexedDB 저장 미구현.
- 제스처 감도 상수들은 SpaceScene 상단 + gestureRecognizer 상단에 모여 있음.
